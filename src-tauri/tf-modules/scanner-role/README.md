# scanner-role

Terraform module that creates `CloudSawScannerRole` — the least-privilege IAM
role CloudSaw assumes when running a scan.

This directory is **read-only at runtime**. The CloudSaw Rust backend
(`src-tauri/src/terraform/`) copies it into a per-account working directory
under `tf-work/{account_id}/` before running `terraform init`/`plan`/`apply`.
Terraform state lives in that per-account working directory **only** — never
in the app bundle, never in the repo. See Contract 05 §Constraints.

## Inputs (all set by CloudSaw, none user-typed)

| Variable | Source | Notes |
|---|---|---|
| `trusted_principal_arn` | `sts:GetCallerIdentity` | Assumed-role sessions are unwrapped to the underlying role ARN. Wildcards rejected by validation. |
| `policy_variant` | UI opt-in | `security_audit` (default) or `read_only_access` (opt-in with warning). |
| `external_id` | Generated by CloudSaw, stored locally | 32+ random chars. Bound to the trust policy as a confused-deputy guard. |
| `role_name` | Constant | `CloudSawScannerRole`. |

## Outputs

| Output | Used by |
|---|---|
| `role_arn` | Contract 06 (`sts:AssumeRole`). |
| `trust_policy_json` | Contract 05 acceptance check ("principal equals caller ARN"). |
| `policy_variant` | UI badge. |

## Safety rules baked in

- One IAM role. One managed-policy attachment. No inline policies. No other
  resources of any kind.
- `Principal: "*"` is impossible — the variable validator blocks it.
- `terraform destroy` is **not** exposed by the app in this contract. Removal
  is out-of-band.
- Provider configuration is empty — credentials come from the parent process's
  AWS SDK provider chain. The module never writes credentials to disk.
